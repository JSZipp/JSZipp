Migration

Upgrade guide

JSZipp can replace JSZip in many browser workflows, but it is not an API-compatible drop-in.

Quick comparison

TaskJSZipJSZipp
Create ZIPzip.file(), generateAsync()writer.add(), writer.close()
Read ZIPJSZip.loadAsync()openZip(source)
Modify existing archiveMutate loaded objectOpen old archive and write a new archive
OutputBrowser and Node-oriented optionsWeb-native Blob, Response, stream, bytes

Create a ZIP Blob

import { ZipWriter } from "web-jszipp";

const writer = new ZipWriter({ outputAs: "blob" });
await writer.add({ path: "images/logo.png", data });
const blob = await writer.close();

Read one file

import { openZip } from "web-jszipp";

const zip = await openZip(file, { pathMode: "strict-package" });
const readme = await zip.get("README.md")?.text();
await zip.close();

Migration dealbreakers

  • No fluent folder object API. Pass full ZIP paths such as images/logo.png.
  • No in-place mutation. Rebuild an archive when removing or replacing entries.
  • Synchronous writing accepts only string, Uint8Array, and ArrayBuffer data.
  • Legacy browser and Node-specific output modes are outside the main target.

See the source maintenance notes in docs/migrate-from-jszip.md for the long-form migration guide.
