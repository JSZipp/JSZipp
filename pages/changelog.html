Unreleased

In development. Breaking changes are still acceptable before the first publish.
  • Added meta.dosAttributes: the MS-DOS attribute byte (low 8 bits of external attributes). Range 0x00–0xff; the directory bit 0x10 must match the entry kind. Allowed for the Dos, Dos | Ntfs, and Dos | Unix | Ntfs timestamp modes; rejected for Dos | Unix.
  • Added explicitDirectoryEntries writer option (on ZipEncoderOptions, default false). When true, parent directories implied by an entry's path are materialized as standalone entries.
  • Changed NTFS timestamp mode no longer requires meta.createdAt / meta.lastAccess; each missing value now defaults to meta.modifiedAt.
  • Changed (breaking) reader entries no longer expose a pre-decoded unixMode. Derive it from externalAttributes >>> 16 (Unix mode) and externalAttributes & 0xff (DOS attributes). Reduces reader bundle size.
  • Changed meta.unixPermissions now requires the Unix timestamp mode (timestamps must include TimestampMode.Unix); otherwise it is rejected.
  • Added option range validation: timestamps must be a bitmask in 0–7; entry timestamps must be valid dates on or after 1970 (negative/NaN rejected); reader maxArchiveSize / maxEntrySize must be non-negative and finite.
  • Docs new "Timestamp Modes and Archive Size" section; clarified that spec-related options belong on ZipEncoderOptions, not ZipWriterOptions.

1.0.0

Initial public documentation baseline. The package currently reports version 1.0.0 in package.json.
  • Browser-focused ZIP writer with Blob, Response, ReadableStream, Uint8Array, and ArrayBuffer output.
  • Random-access ZIP reader through openZip().
  • Strict path handling, strict-package mode, anti-zip-bomb caps, ZIP64, metadata handling, and filename decoding support.
  • TypeScript declarations and dependency-free runtime package.
