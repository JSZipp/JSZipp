How to contribute

Contributions should stay small, testable, and aligned with JSZipp's browser-first ZIP safety goals.

Local workflow

pnpm install
pnpm test
pnpm run typecheck
pnpm run build

Contribution checklist

  • Keep behavior changes focused and document the trust-boundary impact.
  • Add or update tests for writer output, reader parsing, path safety, metadata, or limits as appropriate.
  • Preserve public exception classes and DOMException names.
  • Update public docs when API behavior or migration guidance changes.

Documentation ownership

Public pages should summarize user-facing behavior. Deep implementation rationale belongs in the docs/ markdown references so the public site stays approachable.
