1.0.7 - 2026-06-08

Current package version. This release adds an opt-in Web Worker writer backend, ships worker plugin and worker script subpaths for modern and legacy browser builds, expands migration and API docs, and consolidates the repository's normative documentation under specs/.

Added

  • Added opt-in worker-backed async entry preparation for ZipWriter.add() through createWorkerBackend() in the new web-jszipp/worker-plugin subpath.
  • Added worker-backend controls for lazy worker construction, optional local fallback, byte transfer strategy ("copy", "auto", "transfer"), and a minSize threshold before async entries are offloaded.
  • Added worker script package subpaths so applications can host static worker files under their own CSP and extension rules: web-jszipp/worker-script plus matching browser-legacy/cr61ff58 and browser-legacy/cr86ff68 worker plugin and worker script subpaths.
  • Added the dedicated worker preparation runtime and packaging outputs for modern module workers and legacy classic workers.
  • Added export coverage for the new worker plugin and worker script package subpaths across ESM, CommonJS, UMD, and both legacy browser build families.
  • Added a worker-compression demo page that visualizes main-thread stalls while comparing JSZipp's normal writer path, JSZipp worker-backed writing, fflate, and native compression primitives.
  • Added worker-backend regression coverage for byte-identical output, sync-path isolation, backend reuse, fallback behavior, UTF-8 minSize handling, abort isolation, and package export validation.
  • Added a compile-only regression check for the deprecated worker alias and a worker-backend regression test that simulates a backend returning the wrong path metadata.
  • Added a Playwright browser smoke test that imports the built worker plugin, constructs a real module worker from dist/jszipp.worker.mjs, writes an archive, and validates the result with an independent ZIP reader.
  • Added specs/README.md to define the repository's normative specification set.

Changed

  • Kept the default web-jszipp import worker-free. Worker writing is now an explicit opt-in on ZipWriter through the worker option.
  • Extended the public writer configuration surface with the shared ZipWorkerBackend option while keeping synchronous writer methods on the original local code path.
  • Expanded the API reference, installation guide, examples page, and README with worker plugin usage, worker-source lifecycle guidance, transfer-mode behavior, size-threshold tuning, and compat-build examples.
  • Corrected worker-script documentation to use the actual emitted classic-worker filenames jszipp.worker.js, cr61ff58/jszipp.worker.js, and cr86ff68/jszipp.worker.js.
  • Reworked the upgrade guide into a full JSZip-to-JSZipp migration reference covering architecture differences, dealbreakers, translation examples, safer reader defaults, ZIP64 behavior, and decision guidance.
  • Replaced the standalone migration document with the expanded upgrade guide and updated site navigation and cross-links to point at the new specs/ locations.
  • Moved the repository's behavioral and maintenance documents from scattered docs/ locations into the unified specs/ tree and updated documentation links accordingly.

Fixed

  • Preserved older-browser compatibility for the worker plugin builds by validating the compat outputs and avoiding direct AbortSignal.throwIfAborted() calls in generated legacy worker-plugin bundles.
  • Ensured worker-prepared async entries produce the same ZIP bytes as the normal in-thread writer path for the same inputs and metadata.
  • Kept the deprecated worker alias for workerSource working in the worker plugin so existing integration code keeps running while migrating to the new option name.
  • Fixed the public TypeScript shape of createWorkerBackend() so the deprecated worker alias for workerSource still compiles for TS consumers while the runtime migration path remains unchanged.
  • Hardened ZipWriter's worker backend integration by rejecting prepared entries whose returned path or isDirectory does not match the writer's already-reserved entry path, preventing custom or buggy backends from bypassing duplicate-path and entry-kind invariants.

Compatibility

  • Preserved the public writer and reader API across the default modern build and both legacy browser targets while adding parallel worker plugin entry points.
  • Kept synchronous writeSync() and closeSync() local-only; they do not depend on worker availability and do not route through the worker backend.

Breaking changes

  • No breaking public API changes.

1.0.6 - 2026-06-07

Previous patch release. This release fixed legacy compatibility and safety gaps, strengthened release validation, and updated docs to match the implemented browser, stream, and test behavior.

Fixed

  • Enforced maxEntrySize during legacy raw-DEFLATE inflation, so a ZIP entry with an understated uncompressed-size header cannot fully expand beyond the configured cap before JSZipp detects the problem.
  • Passed the inflated-size cap through the compat DecompressionStream polyfill path while preserving native DecompressionStream behavior in modern builds.
  • Delayed demo object-URL revocation until the next task, avoiding engines and headless automation aborting a download when the URL is revoked in the same tick as the click.
  • Added a demo readiness marker so browser end-to-end tests wait for module bootstrap and event listener binding before interacting with the UI.

Changed

  • Clarified legacy bundle behavior for stream, Blob, File, and Response interop. Compat builds may buffer polyfilled streams before constructing Blob or Response objects to avoid native brand-check corruption on older engines.
  • Updated the benchmark page so JSZipp is labeled as a CDN build when the benchmark runs from the public GitHub Pages origin, and as a local build otherwise.
  • Refactored test scripts so pnpm test runs the Vitest unit suite only, while pnpm run test:all runs the full build, unit, and compatibility smoke flow.
  • Updated prepack to use the full validation path without duplicating compatibility smoke runs.

Added

  • Added CI coverage for install, typecheck, test:all, Playwright browser installation, and test:e2e.
  • Added failure artifact upload after end-to-end tests so Playwright traces and screenshots can be captured when browser validation fails.
  • Added or updated compat smoke coverage for the built CR61FF58 and CR86FF68 legacy bundles.
  • Added a lying-header maxEntrySize regression test for the compat smoke suite.
  • Added direct inflater regression coverage for bounded compat inflation.
  • Added vitest.config.ts so Vitest does not accidentally discover Playwright end-to-end tests.
  • Added browser-compatibility documentation describing the polyfill seam, syntax-target rules, Web Streams ponyfills, DEFLATE inflater fallback, private-symbol installs, and manual verification expectations for legacy browser floors.

Documentation

  • Updated README testing instructions to distinguish the source-level Vitest suite from the built-bundle compatibility smoke tests and Playwright end-to-end browser smoke test.
  • Updated testing documentation to describe what the unit suite, compat smoke test, and Playwright demo test prove, and what they intentionally do not prove.
  • Clarified the modern and legacy browser model: the default build targets modern browser APIs, while legacy subpaths keep the same public API and add targeted compatibility code.
  • Documented legacy memory trade-offs around stream-to-Blob and stream-to-Response conversion.

Compatibility

  • Kept the default modern package entry point focused on native browser primitives and DecompressionStream.
  • Kept legacy package subpaths available for older browser pairs: web-jszipp/browser-legacy/cr61ff58 and web-jszipp/browser-legacy/cr86ff68.
  • Preserved the public API across modern and legacy builds: ZipWriter, ZipTransformStream, openZip, readZipStream, and TimestampMode.

1.0.0 - Initial public documentation baseline

Initial stable baseline. This release established the browser-focused ZIP reader/writer API and the public documentation structure.

Added

  • Browser-focused ZIP writer through ZipWriter, with Blob, Response, ReadableStream, Uint8Array, and ArrayBuffer output shapes.
  • Stream-shaped archive writing through ZipTransformStream.
  • Random-access ZIP reading through openZip(), including entries, get(path), and reusable entry readers.
  • Forward-style archive iteration through readZipStream().
  • Entry helpers for text(), bytes(), arrayBuffer(), and stream().
  • Safe reader defaults that reject path traversal, absolute paths, drive-letter paths, drive-relative paths, backslashes, and NUL-byte paths unless the caller opts into a different path policy.
  • strict-package reader profile for untrusted package-like archives, including duplicate-path and Unicode-normalization collision checks.
  • Configurable anti-zip-bomb limits through maxArchiveSize and maxEntrySize.
  • ZIP64 reading and writing with zip64: "auto", "force", and "off" writer behavior.
  • Store and DEFLATE ZIP entry support, including JSZipp's in-repository raw-DEFLATE encoder and platform or compat inflater paths for reading deflated entries.
  • Archive comments, entry comments, custom extra fields, external attributes, Unix permission bits, DOS attribute bits, DOS timestamps, Extended Timestamp extras, and NTFS timestamp extras.
  • Filename decoding support for UTF-8, CP437, TextDecoder-based legacy encodings, and CRC-verified Info-ZIP Unicode Path extra fields.
  • Abort and progress hooks for long-running writer and reader work.
  • TypeScript declarations and a dependency-free runtime package.
  • Default namespace export JSZipp plus named exports for the public runtime API.
  • Browser-oriented documentation pages for overview, installation, FAQ, API reference, examples, performance and limitations, upgrade guidance, bug reporting, contribution, and sponsorship.
  • Interactive browser demo and benchmark pages.

Compatibility baseline

  • Modern build for browser APIs including ReadableStream, TransformStream, Blob, AbortSignal, TextEncoder, TextDecoder, and native DecompressionStream for deflated reads.
  • Legacy browser subpaths for older browser pairs that need additional syntax targeting and runtime polyfills while preserving the same public API.
  • Production bundles keep observable exception categories stable while shortening error messages to compact codes such as E_PATH, E_LIMIT, E_STRUCTURE, E_ZIP64, and E_UNSUPPORTED.

Release note policy

  • Keep this page tied to shipped package versions. Do not leave completed release work under Unreleased after bumping package.json.
  • Prefer categories in this order when relevant: Security, Fixed, Changed, Added, Documentation, Compatibility, and Breaking changes.
  • Call out public API breaks explicitly. Internal test, CI, and documentation changes should still be listed when they affect release confidence or user-facing guidance.
