# JSZipp Specifications

This directory contains the repository's normative specifications.

All normative specification documents MUST live in this directory. No other
folder is allowed to define or own JSZipp behavioral requirements,
compatibility promises, validation rules, or test requirements.

Unless a file explicitly says otherwise, the keywords `MUST`, `MUST NOT`,
`SHOULD`, `SHOULD NOT`, and `MAY` are normative in the RFC 2119 sense.

This repository now stores all project documentation in `specs/`. Some files
describe runtime behavior, some describe implementation constraints, some define
documentation governance, and some define contributor procedures. They are all
treated as normative repository specifications.
