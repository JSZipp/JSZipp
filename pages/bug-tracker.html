Bug tracker

Use the project issue tracker for reproducible bugs, compatibility gaps, security-hardening discussions, and documentation corrections.

Open GitHub issuesRepository

Useful reports include

  • JSZipp version, browser version, and operating system.
  • A minimal archive or script that reproduces the behavior.
  • The expected result and the actual exception name or output.
  • Whether the input archive is trusted, user-uploaded, or generated by another tool.
Security reports: avoid publishing exploit archives or sensitive samples in a public issue. Start with a minimal description and ask for a private handoff path.
